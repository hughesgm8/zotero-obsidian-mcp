## [session] Import Zotero tags into note frontmatter

- **Files Modified**: `src/types.ts`, `src/paper-importer.ts`
- **Change**: Tags assigned to a paper in Zotero now appear in the frontmatter of the imported note.
- **Logic Update**:
  ```
  ZoteroSource gains optional tags[] field.

  fetchItemTags(key):
    GET /api/users/0/items/{key} via Node http
    return item.data.tags[].tag

  importPaper():
    if source.tags is empty → source.tags = await fetchItemTags(source.key)

  buildTags(source):
    [itemType, "literature", "ai-imported", ...source.tags (deduplicated)]

  getRecentItems(): extract tags from Zotero API response
  parseMetadata(): extract tags from JSON path and "**Tags:**" markdown line
  ```
- **Reason**: Tags were never captured from Zotero — `ZoteroSource` had no `tags` field and `buildTags()` only wrote static values. Fix uses the local Zotero API as the authoritative source since MCP metadata output does not include tags.
