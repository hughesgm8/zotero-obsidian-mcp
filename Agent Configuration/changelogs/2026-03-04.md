## [~Session] Add 'Insert AI summary into active note' command

### Change 1
- **File Modified**: `src/paper-importer.ts`
- **Change**: Added `generateSummaryMarkdown(source)` — same LLM pipeline as `importPaper()` but returns summary sections as a markdown string (no frontmatter, no abstract) with an attribution line
- **Logic Update**:
  ```
  fetch full text (up to 40k chars)
  call LLM with same prompt as importPaper()
  parse sections
  return: attribution line + Summary + Takeaways + Questions [+ Relevance]
  ```
- **Reason**: Needed a way to get just the AI content without the full note scaffolding, for inserting into existing notes.

### Change 2
- **File Modified**: `src/import-modal.ts`
- **Change**: `ImportModal` now accepts optional `title` and `onSelect` callback; if `onSelect` is provided it runs instead of the default create-note behaviour
- **Logic Update**:
  ```
  constructor(app, importer, options?: { title?, onSelect? })
  selectPaper():
    if onSelect provided → call onSelect(source)
    else → existing importPaper() flow
  ```
- **Reason**: Makes the modal reusable for both commands without duplicating the search/display logic.

### Change 3
- **File Modified**: `src/main.ts`
- **Change**: Added "Insert AI summary into active note" command using `editorCallback` (auto-hidden when no editor is active); opens modal with custom `onSelect` that calls `generateSummaryMarkdown()` and inserts at cursor via `editor.replaceSelection()`
- **Reason**: Users with existing Zotero Integration notes can enrich them with AI summaries without creating a duplicate file.

## [~Session] Fix text selection and copy-with-sources in chat view

### Change 1
- **File Modified**: `styles.css`
- **Change**: Added `user-select: text` and `cursor: text` to `.zotero-chat-message-content`
- **Logic Update**:
  ```
  .zotero-chat-message-content {
    + user-select: text
    + cursor: text
  }
  ```
- **Reason**: Obsidian applies `user-select: none` broadly to sidebar UI elements. Without overriding it, users couldn't highlight or copy text from their own message bubbles.

### Change 2
- **File Modified**: `src/chat-view.ts`
- **Change**: Copy button on assistant messages now includes the sources list in the clipboard output
- **Logic Update**:
  ```
  on copy click:
    text = msg.content
    if msg.sources:
      text += "\n\n**Sources:**\n"
      for each source: text += "- title — authors (year)\n"
    clipboard.write(text)
  ```
- **Reason**: `msg.content` is only the LLM prose body — sources are stored separately in `msg.sources`. Previously clicking copy omitted them entirely, which was frustrating when pasting into notes.
