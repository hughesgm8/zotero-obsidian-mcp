# Changelog — 2026-03-10

## [Session] CONVERSATION HISTORY & SAVE UX — v0.2.2

### Save icon update
- **File Modified**: `src/chat-view.ts`, `styles.css`
- **Change**: Save button icon changed from `file-output` to `download` (down-arrow-into-tray), matching Copilot's visual convention. More visually distinct from the "New chat" (`square-pen`) icon.
- **Reason**: User lost a conversation by clicking "New chat" instead of "Save" because the icons were too similar.

---

### Persistent chat history with sidebar popover
- **Files Modified**: `src/types.ts`, `src/main.ts`, `src/chat-view.ts`, `styles.css`
- **Change**: Conversations are now automatically cached to plugin data whenever the user clicks "New chat" or closes the chat view. A history button (clock icon) in the controls bar opens a non-disruptive popover anchored above the button.
- **Logic Update**:
  ```
  // New CachedChat interface: { id, title, messages, cachedAt, savedFilePath? }
  // Plugin data restructured: { settings, chatHistory[] }  ← backward-compat fallback for old flat format
  // clearChat() → addToChatHistory(messages) → saveChatHistory() → Notice("Chat saved to history")
  // onClose() → addToChatHistory(messages) if messages.length > 0
  // History button → toggleHistoryPopover(anchor)
  //   showHistoryPopover(): position: fixed div appended to document.body
  //     bottom = innerHeight - anchor.top + 6px; right = innerWidth - anchor.right
  //     click-outside listener (deferred via setTimeout to avoid self-close)
  //   Popover list items: title + date, hover-reveal actions
  //     Open source file → saveConversation(entry.messages) if no savedFilePath, then openFile()
  //     Edit title → inline input replace, commit on Enter/blur, cancel on Escape
  //     Delete → first click expands button to "Delete?" pill; second click confirms; 3s timeout resets
  // pruneExpiredChatHistory(): filters entries older than 30 days; called on plugin load + popover open
  ```
- **Reason**: User accidentally cleared a conversation with no way to recover it. Design modelled on Obsidian Copilot's `ChatHistoryPopover` (Radix UI `side="top" align="end"`). Obsidian's native `Modal` was tried first but rejected — it opens center-screen, which is jarring for functionality confined to the sidebar. Fixed popover appended to `document.body` avoids sidebar `overflow: hidden` clipping.

---

### Delete confirmation UX
- **File Modified**: `src/chat-view.ts`, `styles.css`
- **Change**: First delete click expands the trash icon button into a solid red "Delete?" pill. Second click within 3 seconds confirms deletion. After 3 seconds, button resets to the icon automatically.
- **Logic Update**:
  ```
  deleteBtn.click #1 → addClass("delete-confirm") → empty() → setText("Delete?") → setTimeout(reset, 3000)
  deleteBtn.click #2 → deleteChatHistoryEntry(id) → row.remove()
  ```
- **Reason**: The previous two-click approach (icon silently turns red) gave no feedback that a second click was needed. Expanding to a text label is self-explanatory.
