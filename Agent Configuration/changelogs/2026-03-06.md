## [Session] PERSISTENT IMPORT LOADING NOTICE (Issue #3)

### Insert AI summary into active note (`src/main.ts`)
- **File Modified**: `src/main.ts`
- **Change**: Replaced the auto-dismissing 8-second Notice with a persistent one that stays until the import finishes.
- **Logic Update**:
  ```
  // Before
  new Notice(`Generating summary...`, 8000)
  markdown = await generateSummaryMarkdown(source)
  editor.replaceSelection(markdown)
  new Notice("Summary inserted.")

  // After
  loadingNotice = new Notice(`Generating summary...`, 0)  // 0 = stays until dismissed
  try:
    markdown = await generateSummaryMarkdown(source)
    editor.replaceSelection(markdown)
    loadingNotice.hide()
    new Notice("Summary inserted.")
  catch err:
    loadingNotice.hide()
    throw err  // re-thrown so outer catch in selectPaper shows error notice
  ```
- **Reason**: The old 8-second notice could disappear while the LLM was still generating (20–30s). Users had no signal the operation was still running, potentially causing them to think it had failed or trigger it again. The `editor` reference is captured at command invocation time and remains valid even if the user navigates away, so the insert always goes to the correct note.

### Import paper — create new note (`src/import-modal.ts`)
- **File Modified**: `src/import-modal.ts`
- **Change**: Same persistent notice pattern applied to the "create new note" import path for consistency.
- **Logic Update**:
  ```
  // Before
  new Notice(`Importing "${title}"...`, 8000)
  try:
    result = await importPaper(source)
    new Notice(`Imported: ${title}`)
    open note
  catch:
    new Notice(`Import failed: ${msg}`, 10000)

  // After
  loadingNotice = new Notice(`Importing "${title}"...`, 0)
  try:
    result = await importPaper(source)
    loadingNotice.hide()
    new Notice(`Imported: ${title}`)
    open note
  catch:
    loadingNotice.hide()
    new Notice(`Import failed: ${msg}`, 10000)
  ```
- **Reason**: Consistency with the insert path; user sees a persistent indicator for the full operation duration.

## [Session] RECENTLY ADDED PAPERS IN IMPORT MODAL (Issue #2)

- **Files Modified**: `src/paper-importer.ts`, `src/import-modal.ts`, `styles.css`
- **Change**: The Smart Import modal now shows the 10 most recently added Zotero papers immediately on open, before the user types anything. If the user clears the search box after typing, the recent list reappears. A styled "Recently added" section label separates the default state from search results.
- **Logic Update**:
  ```
  ImportModal.onOpen():
    render search input
    call loadRecentItems()               # new: populate on open

  ImportModal (search input listener):
    if query is empty → loadRecentItems()  # was: clear results
    else → debounced runSearch(query)      # unchanged

  ImportModal.loadRecentItems():
    show "Loading recent papers..."
    results = importer.getRecentItems(10)
    render "Recently added" section label
    render each result via renderResult()

  PaperImporter.getRecentItems(limit):
    GET http://localhost:23119/api/users/0/items/top
      ?sort=dateAdded&direction=desc&limit={limit}
    via Node http module (not requestUrl — avoids Zotero CSRF rejection)
    filter out attachments and notes
    map to ZoteroSource[] using existing formatAuthors() helper

  styles.css:
    add .zotero-import-section-label (small caps, faint, uppercase label)
  ```
- **Reason**: Replaces blank modal state with immediately useful content. Applies to both Smart Import variants (new note and insert into active note). Uses the Node `http` module — the same approach as the MCP client — because Obsidian's `requestUrl` triggered a 403 CSRF rejection from Zotero's local API; the Node http module sends no Origin header and is accepted. The Zotero local API requires no API key in local mode.
