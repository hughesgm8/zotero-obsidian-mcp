## [Session] RECENTLY ADDED PAPERS IN IMPORT MODAL (Issue #2)

- **Files Modified**: `src/paper-importer.ts`, `src/import-modal.ts`, `styles.css`
- **Change**: The Smart Import modal now shows the 10 most recently added Zotero papers immediately on open, before the user types anything. If the user clears the search box after typing, the recent list reappears. A styled "Recently added" section label separates the default state from search results.
- **Logic Update**:
  ```
  ImportModal.onOpen():
    render search input
    call loadRecentItems()               # new: populate on open

  ImportModal (search input listener):
    if query is empty → loadRecentItems()  # was: clear results
    else → debounced runSearch(query)      # unchanged

  ImportModal.loadRecentItems():
    show "Loading recent papers..."
    results = importer.getRecentItems(10)
    render "Recently added" section label
    render each result via renderResult()

  PaperImporter.getRecentItems(limit):
    GET http://localhost:23119/api/users/0/items/top
      ?sort=dateAdded&direction=desc&limit={limit}
    via Node http module (not requestUrl — avoids Zotero CSRF rejection)
    filter out attachments and notes
    map to ZoteroSource[] using existing formatAuthors() helper

  styles.css:
    add .zotero-import-section-label (small caps, faint, uppercase label)
  ```
- **Reason**: Replaces blank modal state with immediately useful content. Applies to both Smart Import variants (new note and insert into active note). Uses the Node `http` module — the same approach as the MCP client — because Obsidian's `requestUrl` triggered a 403 CSRF rejection from Zotero's local API; the Node http module sends no Origin header and is accepted. The Zotero local API requires no API key in local mode.
